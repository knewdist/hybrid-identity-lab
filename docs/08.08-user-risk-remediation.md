# 08.8 — User Risk Remediation (Require Password Reset for High-Risk Users)

## Objective

Implement a **Conditional Access policy** that forces users to reset their password when Microsoft Entra detects a **high-risk user account**.

This demonstrates **automatic remediation of potentially compromised identities** using Microsoft Entra Identity Protection and Conditional Access.

This is a core **Zero Trust identity protection control**.

---

# Environment

Tenant:

simmonslab.onmicrosoft.com

Identity platform:

Microsoft Entra ID

Security components used:

* Identity Protection
* Conditional Access
* Multi-Factor Authentication
* Privileged Identity Management (PIM)

---

# Background

Microsoft Entra **Identity Protection** continuously analyzes authentication activity and assigns risk levels to user accounts.

Examples of signals that can trigger user risk:

* Leaked credentials detected on the dark web
* Password spray attacks
* Credential stuffing attempts
* Suspicious authentication patterns
* Malware-associated IP infrastructure

When a user is considered **high risk**, the system can enforce **automatic remediation policies**.

---

# Identity Risk Response Flow

Authentication and remediation workflow:

```
User login
     │
     ▼
Microsoft Entra authentication
     │
     ▼
Identity Protection evaluates account risk
     │
     ▼
High-risk user detected
     │
     ▼
Conditional Access policy triggered
     │
     ▼
User forced to reset password
     │
     ▼
Access restored
```

This automatically **contains compromised credentials**.

---

# Step 1 — Navigate to Conditional Access

Open:

```
Microsoft Entra ID
↓
Protection
↓
Conditional Access
↓
New Policy
```

---

# Step 2 — Create Policy

Policy Name:

```
Require Password Reset for High-Risk Users
```

---

# Step 3 — Assign Users

Assignments:

```
Users or agents
↓
All users
```

In enterprise environments, administrators typically **exclude break-glass emergency accounts**.

---

# Step 4 — Target Resources

Target resources:

```
All resources
(formerly "All cloud apps")
```

This ensures the policy applies across all protected Microsoft cloud services.

---

# Step 5 — Configure Risk Condition

Conditions:

```
User risk
```

Configuration:

```
High
```

This means the policy activates only when Microsoft Entra determines that the **user account is highly likely to be compromised**.

---

# Step 6 — Configure Access Control

Access Controls:

```
Grant access
↓
Require password change
```

When triggered, the user must **reset their password before access is granted**.

---

# Result

Authentication now includes automated compromise remediation:

```
User login
↓
Identity Protection detects compromised account
↓
Conditional Access policy triggered
↓
Forced password reset
↓
User regains access with new credentials
```

---

# Security Impact

This control protects organizations against:

* Credential leaks
* Stolen passwords
* Password spray attacks
* Account takeover attempts
* Persistent compromised credentials

By forcing a password reset automatically, the system quickly **neutralizes compromised accounts**.
