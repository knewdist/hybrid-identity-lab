# 08.4 — Conditional Access: Require MFA for Administrators

## Objective

Implement a **Conditional Access policy** in Microsoft Entra ID to require **multi-factor authentication (MFA)** for administrator accounts.

This protects privileged identities and demonstrates a **core Zero Trust identity security control**.

---

## Environment

**Tenant**


simmonslab.onmicrosoft.com


**Identity Platform**


Microsoft Entra ID


**Groups Used**


Cloud-Admins
Cloud-Users


---

## Background

Microsoft Entra tenants often begin with **Security Defaults enabled**.

Security Defaults automatically enforce baseline protections such as:

- MFA for administrators
- Blocking legacy authentication
- Basic identity protection

However:


Security Defaults cannot be used together with Conditional Access.


To create **custom identity security policies**, Security Defaults must first be disabled.

---

# Step 1 — Disable Security Defaults

Navigate to:


Microsoft Entra ID
→ Properties
→ Manage security defaults


Change:


Security defaults → Disabled


This allows the tenant to use **Conditional Access policies** instead of automatic defaults.

---

# Step 2 — Open Conditional Access

Navigate to:


Microsoft Entra ID
→ Protection
→ Conditional Access


Create a **new policy**.

---

## Screenshot

![Conditional Access Dashboard](../screenshots/08.4/conditional-access-dashboard.png)


Conditional Access Dashboard


---

# Step 3 — Configure the Conditional Access Policy

**Policy name**


Require MFA for Admin Accounts


**Assignments**


Users → Cloud-Admins group


**Target resources**


All cloud apps


**Access controls**


Grant access
Require multi-factor authentication


**Policy state**


Enabled


---

# Authentication Flow

Once the policy is enabled, administrator authentication follows this flow:


cloud-admin login
│
▼
Microsoft Entra ID
│
Conditional Access evaluation
│
Require MFA
│
Access granted


---

# Security Impact

This policy protects privileged accounts against:

- Password-only attacks
- Credential stuffing
- Phishing-based compromise

Requiring MFA for administrators is one of the **most widely recommended identity protections in enterprise environments**.
